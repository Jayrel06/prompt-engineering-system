## Integration Points

### With Prompt Engineering System

The MCP server integrates with:

1. **scripts/context-loader.py**
   - Called by assemble_context tool
   - Assembles context based on mode
   - Returns formatted prompts

2. **frameworks/**
   - Listed by list_frameworks tool
   - Retrieved by get_framework tool
   - Organized by category

3. **templates/**
   - Listed by list_templates tool
   - Retrieved by get_template tool
   - Organized by category

4. **context/**
   - Searched by search_knowledge tool
   - Captures saved to context/learnings/captures/
   - Referenced by context assembly

### With Claude Desktop

The server communicates via:
- **Protocol:** Model Context Protocol (MCP)
- **Transport:** stdio (JSON-RPC over stdin/stdout)
- **Configuration:** claude_desktop_config.json
- **Lifecycle:** Started/stopped with Claude Desktop

## Development Workflow

### Making Changes

1. Edit `src/index.ts`
2. Run `npm run build` or `npm run watch`
3. Restart Claude Desktop to load changes
4. Test with Claude Desktop

### Adding Tools

1. Add tool definition to TOOLS array
2. Add case to switch statement in CallToolRequestSchema handler
3. Implement tool logic
4. Update documentation (README.md, USAGE_EXAMPLES.md)
5. Update CHANGELOG.md
6. Build and test

### Testing

1. Run `npm test` for setup verification
2. Test manually in Claude Desktop
3. Check error handling
4. Verify documentation matches behavior

## Maintenance

### Regular Updates

- Update dependencies: `npm update`
- Check MCP SDK updates: Check @modelcontextprotocol/sdk releases
- Update documentation for new features
- Update CHANGELOG.md for each version

### Troubleshooting Files

If issues occur, check these files in order:

1. `npm test` output - Setup verification
2. `dist/index.js` - Verify compiled output exists
3. `package.json` - Verify dependencies
4. `claude_desktop_config.json` - Verify configuration
5. Claude Desktop logs - Application errors
